# momit.fm Episode Style Guide

スキルが参照する「秘伝のタレ」知識ベース。RSS 実データから抽出。

---

## タイトルパターン

### フォーマット
```
{N}. {Topic1} / {Topic2} / {Topic3}
```

### ルール
- トピック区切りは ` / `（半角スペース + スラッシュ + 半角スペース）
- トピック数は 2-4 個（3 個が最多）
- 関連概念の結合には `×` を使う場合あり（例: `YouTube×創作欲の相性`）
- 絵文字は使わない
- 全体で 120 文字以内目安
- カジュアル表現と具体的なプロダクト/サービス名をバランスよく混ぜる
- 子育て系トピックと技術系トピックのバランスを意識

### パターン傾向
- 疑問形を含むトピック: `自動化orAI?使い分けの勘所`、`宿題ゼロの夏休みはアリかナシか`
- 「の」で具体化: `マイクラの隠れた教育効果`、`Nano Bananaの日本語精度`
- 体験・実践を示す語: `活用術`、`体験談`、`レビュー`、`検証`、`チャレンジ`
- 固有名詞はそのまま英語: `Claude Code`、`NotebookLM`、`DORA AI Capabilities Model`

## Shownote（Description）構造

### フォーマット
```
[冒頭サマリー 2-3 文。エピソード全体のテーマを自然な日本語で要約]

[絵文字] **[太字トピック]** – [説明文]
[絵文字] **[太字トピック]** – [説明文]
...

…………………………………………………………………

フィードバックは #momitfm (https://twitter.com/search?q=%23momitfm) もしくは おたよりフォーム (https://note.com/kamiyuuu/n/n5cb3cd154259) より募集しています。

Yu Kamiya (https://twitter.com/kamiyuuu) / miho (https://twitter.com/miholovesq)

momit hub (https://hub.momit.fm/) / YouTube (https://www.youtube.com/@momitfm)
```

### ルール
- **冒頭サマリー**: 2-3 文。聞きどころのハイライトを自然な会話調で紹介
- **トピック**: 絵文字 + 太字タイトル + em dash (–) + 説明
- **トピック数**: メインテーマ 3-5 個 + サブトピック数個（合計 5-12 個）
- **絵文字の選択**: 内容に合った絵文字を使う（技術: 🤖🔧🧰💻、子育て: 📝✈️🎮、概念: 🧠📏🌐）
- **説明文**: 1 文で簡潔に。会話的なトーンで内容のエッセンスを伝える
- **区切り線**: `…………………………………………………………………`（全角三点リーダー × 15）
- **フォーマット**: Plain text（HTML タグなし）。コピペしやすさ重視
- **リンク形式**: `テキスト (URL)` のインライン形式

✂️ **編集は"人の耳"を残す** – フィラー除去や無音整形はRiversideまかせ、カット判断は人。

🧠 **コンテキストエンジニアリング** – 目的→仕様→手順→検証の情報を揃えると、AI・エージェントの安定度が段違い。
📏 **使い分けの基準** – 定型はオートメーション、文章生成/要約/探索は生成AI、品質責任が高い箇所は人で締める。

## 広告挿入ポイントの基準

- エピソード全体を 3 等分した区切り付近
- トピック転換の自然なブレイクポイント
- アイスブレイク → メイントピック移行の間（前半）
- メイントピック間の転換点（中盤）
- 2-3 箇所が目安
